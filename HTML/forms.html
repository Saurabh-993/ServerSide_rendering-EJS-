Important Concept


name & value Attribute

so you have to understand one thing that name attribute is very important, as through it the data is transferred to the backend.

Basically, whenever we submit a form the browser sends the name&value pair to the server (Backend), similar to the key-value pair. 

      
        <input type="text" name="username" value="Rohan">
        <input type="password" name="pass" value="12345">

So after submit, the server will get it.

      
        username=Rohan&pass=12345

Note:-If you didn't put the name attribute than server will not get the information and browser will ignore it.


name & value with checker-box & radio

whenever you put the name & value attribute in input tags with other attributes like checker-box & radio what actually happens is a bit tricky thing.

suppose you put code like this:-

      
        <input type="checkbox" name="terms" checked>

then browser by default will put the value ="on" (where there is no "value" attribute is set by the user) i.e. 

      
        terms=on

but when we put the values also,

      
        <input type="checkbox" name="skills" value="HTML" checked>
        <input type="checkbox" name="skills" value="CSS" checked>

then after submitting, the Server will know the values, or you can say that the Backend will know about the checkboxes that got ticked✅. 

      
        skills=HTML&skills=CSS

so, In the table below we will discuss when do we need the value attribute.


when do we need value attribute
Situation Without "value" With "value="HTML""
Single checkbox: "Terms accept kiye?" terms=on aayega. Kaam chal jayega kyunki bas haan/na check karna hai Zaroorat nahi
Multiple checkbox: Skills chunni hain skills=on&skills=on aayega. Backend ko ghanta nahi pata chalega ki kaunsi skill thi skills=HTML&skills=CSS aayega. Ab backend ko exact pata hai
Radio: Gender chunna hai gender=on aayega chahe Male select karo ya Female. Dono ka same bhejega. Bekar gender=male ya gender=female aayega. Perfect

Example of radio:-

First, we will discuss the wrong way:

      
        <input type="radio" name="gender"> Male
        <input type="radio" name="gender"> Female

This is the wrong way , because here we didn't provide any value attribute.

So here whatever you select It doesn't matter, as the Server will always get "gender = on" Server will never know that which one got selected.


Now, we will discuss the right way:

      
        <input type="radio" name="gender" value="male"> Male
        <input type="radio" name="gender" value="female"> Female

This is the right way , As here we have provided the value attribute.

Now here, Server will get either "gender = male" or "gender = female".


In the easiest way we can understand the whole discuss point as:-

  1. name:-Form ka data backend me bhejne ke liye compulsory hai. Ye "key" banata hai.
  2. value:-Server ko kya value bhejni hai vo decide karta hai.
    • Text, Email, Password mai: User jo type karega wahi value ban jayegi. Isliye yaha value="" likhne ki zaroorat nahi.
    • Checkbox, Radio, Option me: User kuch type nahi karta, sirf select karta hai. To server ko kya bhejna hai vo tu value="" me batayega.

      Nahi batayega to browser apne se on bhej dega jo 99% cases me bekar hai.

Ek line me: name batata hai "kaunsa dabba hai", value batata hai "dabbe ke andar kya hai".
